This list is intentionally conservative. It should be expanded before any academic submission, investor memo, or public technical claim.

## Research Areas To Cite Before Publication

The whitepaper should eventually cite peer-reviewed work in:

- Working memory capacity and chunking
- Attention and cognitive load
- Human-computer interaction for interruption-sensitive systems
- Retrieval-augmented generation
- Speech neuroprostheses
- Assistive and augmentative communication
- Brain-computer interface control paradigms
- Neurotechnology ethics and mental privacy

## Claim Hygiene

Use careful language:

- Say "BCI-compatible future direction," not "BCI product."
- Say "assistive communication concept," not "medical treatment."
- Say "non-invasive prototype," not "neural implant."
- Say "context-aware cues," not "thought insertion."
- Say "cognitive load sensing exploration," not "mind reading."

## Open Research Questions

1. How short must a cue be to help rather than interrupt?
2. When should the system choose silence?
3. How should confidence be displayed in audio-only contexts?
4. Can haptic gestures control depth without increasing cognitive load?
5. Which assistive communication scenarios benefit most from context-aware candidate generation?
6. What privacy architecture is required for personal memory retrieval?
7. How should the system be evaluated across usefulness, agency, accuracy, and interruption cost?
