# SYMBORG Device Family Roadmap

SYMBORG should not be framed as one device. It should be framed as a family of symbiotic cognitive interfaces, moving from realistic wearable products to future BCI-compatible research systems.

## Device family overview

```text
SYMBORG App      -> phone/screen prototype
SYMBORG Ear      -> private audio whisper
SYMBORG Ring     -> haptic control and silent commands
SYMBORG Glass    -> AR peripheral cognitive cards
SYMBORG Band     -> physiological/cognitive load awareness
SYMBORG Assist   -> assistive communication terminal
SYMBORG Neural   -> future BCI-compatible research layer
```

## 1. SYMBORG App

**Type:** software-first prototype.

**Purpose:** prove the cognitive packet loop before custom hardware.

Inputs:

- live/recorded transcript
- screen context
- calendar and notes
- manual user prompt

Outputs:

- phone screen cards
- desktop overlay
- exportable packet logs

Why first:

```text
Fastest to build.
No hardware dependency.
Best for demos and evaluation.
```

## 2. SYMBORG Ear

**Type:** discreet audio feedback device.

**Hardware form:** bone-conduction earpiece, open-ear earbud, or future custom earpiece.

Output:

```text
quiet cognitive whisper
5-14 word cues
single idea per cue
```

Best use cases:

- social conversation
- meeting support
- language learning
- low-vision navigation
- assistive communication confirmation

Design rule:

```text
The user must feel accompanied, not controlled.
```

## 3. SYMBORG Ring

**Type:** silent command and haptic feedback device.

Inputs:

```text
single tap        -> shorter cue
double tap        -> deeper cue
long press        -> silence
swipe up          -> suggest question
swipe down        -> summarize
side swipe        -> source/confidence
```

Outputs:

```text
one pulse         -> packet ready
two pulses        -> low confidence
long soft pulse   -> suppressed / silence mode
rapid pulse       -> urgent assistive alert
```

Why important:

```text
It gives the user agency without requiring speech or screen interaction.
```

## 4. SYMBORG Glass

**Type:** AR peripheral display.

Inputs:

- front camera
- gaze direction if available
- scene understanding
- optional screen mirroring

Outputs:

- one-line peripheral cards
- confidence iconography
- object labels
- selected memory cue
- assistive candidate grid

Design rules:

```text
Do not cover faces.
Do not center text unless requested.
Do not create social creepiness.
Default to peripheral display.
```

Best use cases:

- meetings
- education
- navigation
- technical work
- assistive context awareness

## 5. SYMBORG Band

**Type:** physiology-aware adaptation layer.

Signals:

- heart rate
- HRV
- respiration estimate
- skin conductance if available
- motion
- optional EEG/fNIRS in research versions

Purpose:

```text
Not thought reading.
Load awareness.
```

Use:

```text
high load   -> reduce output
low load    -> allow deeper packets
stress      -> shorter, calmer cues
speech mode -> suppress audio
```

## 6. SYMBORG Assist

**Type:** assistive communication interface.

Target user:

- limited speech
- limited movement
- one-switch input
- eye tracking input
- future BCI input

Output:

```text
2-4 context-aware sentence candidates
confirm / cancel / speak
```

Example:

```text
Doctor: How is the pain?

SYMBORG candidates:
1. It is better now.
2. My neck hurts more at night.
3. I need to change position.
```

This is the strongest serious demo because it connects SYMBORG to autonomy, dignity, and communication.

## 7. SYMBORG Neural

**Type:** future research and BCI-compatible output layer.

This is not an early consumer claim.

Possible tracks:

- BCI input: user intent selection
- visual cortex output research: low-resolution phosphene patterns
- auditory/speech neuroprosthesis integration
- somatosensory stimulation for tactile codes
- closed-loop calibration and safety

Positioning:

```text
SYMBORG does not need to build the implant.
SYMBORG can become the cognitive UX layer that sits above implants.
```

## Product sequence

```text
Phase 0: App prototype
Phase 1: Ear + Ring
Phase 2: Glass + local memory vault
Phase 3: Band / load awareness
Phase 4: Assistive communication terminal
Phase 5: BCI-compatible research integrations
Phase 6: Neural-write clinical research concepts
```

## The brand distinction

```text
Cyborg adds machine to body.
SYMBORG adds symbiosis to thought.
```
