## LinkedIn Post: Live Demo Launch

```text
I have published the first working local demo for SYMBORG.

SYMBORG is a concept for a cognitive front-end layer between AI retrieval systems and human attention.

The prototype shows a simple loop:

Transcript -> cognitive packets -> quality gate -> output route.

Instead of returning a long generated answer, the system produces short thought seeds:

- a question
- an anchor
- a counterpoint
- a selectable assistive sentence candidate

Each packet is scored for brevity, actionability, confidence, interruption risk, and cognitive load. Then the router decides whether it should appear as a screen card, audio whisper, AR card, haptic cue, assistive candidate, or stay silent.

The most serious use case is assistive communication: helping a constrained-input user select meaningful sentence candidates from context while preserving agency and confirmation.

This is not a medical device claim. It is not mind reading, and it is not affiliated with any BCI company. The first direction is deliberately non-invasive and interface-focused.

I would value critique from people working in HCI, wearable AI, accessibility, assistive communication, neurotechnology ethics, or brain-computer interfaces.

Repository:
https://github.com/Pharmacist65/symborg
```

## LinkedIn Post: Main Launch

```text
I am starting to publish a project I have been developing: SYMBORG.

SYMBORG is a concept for a cognitive front-end layer between AI systems and human attention.

The idea is simple:

AI can retrieve and compress information faster than a person can hold it in working memory. But returning too much information at the wrong time creates noise, overload, and loss of agency.

So the real interface question is not only "Can AI know more?"

It is:

How do we return the right piece of information to a person at the right moment, in the right amount, without replacing their thought?

SYMBORG explores that layer through:

- Context detection
- Local personal memory
- Retrieval
- A Cognitive Packet Engine
- Confidence and ethics filtering
- Subtle output through audio, AR, haptics, or assistive communication UI

I am intentionally starting with a non-invasive prototype direction: a contextual whisper system that turns live conversation into short cognitive cues, not full generated answers.

The more serious long-term use case is assistive communication for constrained-input users: helping a person select meaningful sentence candidates from minimal input while preserving agency and dignity.

This is not a medical device claim, and it is not affiliated with any BCI company. It is an early product/research architecture for a missing human-AI interaction layer.

I would value critique from people working in HCI, wearable AI, neurotechnology ethics, assistive communication, accessibility, or brain-computer interfaces.

Repository: https://github.com/Pharmacist65/symborg
```

## Comment Reply Templates

### If someone asks whether this is a BCI product

```text
Not in the current form. The near-term prototype is non-invasive and uses conversation/screen context plus user-controlled signals. BCI compatibility is treated as a future research direction, especially for assistive communication, not as an early product claim.
```

### If someone asks what is actually working

```text
The repo includes a local browser demo that takes a transcript, generates scored cognitive packets, applies a conservative delivery gate, and routes outputs to screen/audio/AR/haptic/assistive modes. It is heuristic today, but the interface loop is working.
```

### If someone asks how they can help

```text
The most useful help right now would be critique on the interface contract: when should the system speak, when should it stay silent, and how should assistive sentence candidates preserve user agency?
```
