# Neural-Write Research Track

This document explores the most ambitious SYMBORG output direction: returning information by stimulating neural systems directly.

This is not part of SYMBORG v0/v1. It is a future clinical/research track.

## Core question

```text
Can SYMBORG show information inside perception rather than on an external display?
```

The answer is nuanced:

- **In principle:** yes, stimulation of sensory cortex can evoke percepts.
- **Today:** the output is not like a normal computer screen.
- **Near-term realistic target:** symbolic low-bandwidth percepts, not rich images.

## Visual cortex output

The visual cortex can be electrically stimulated to produce **phosphenes**: perceived spots, flashes, shapes, brightness changes, or simple patterns.

A future cortical visual output system might work like this:

```text
camera / AI scene / cognitive packet
        |
        v
visual encoding model
        |
        v
stimulation pattern
        |
        v
phosphene field
        |
        v
user learns the artificial visual language
```

## Why it is not currently an internal screen

A normal screen has millions of pixels, stable colors, high refresh rate, wide field of view, and predictable geometry.

A cortical visual prosthesis has to deal with:

- limited electrode count
- biological variability
- phosphene size and shape variation
- uncertain spatial mapping
- limited field of view
- stimulation safety limits
- long-term tissue response
- calibration burden
- user learning

Therefore, SYMBORG should not say:

```text
We will project high-resolution images into the brain.
```

It should say:

```text
Future SYMBORG Neural may explore low-bandwidth perceptual symbols and phosphene glyphs as a neural output language.
```

## SYMBORG visual cortex strategy

Do not try to recreate an iPhone screen in the brain.

Create a **neural symbolic language**.

Examples:

```text
single dot      -> packet ready
two dots        -> choose option A/B
left arrow      -> direction cue
bright pulse    -> warning
small line      -> object boundary
three pulses    -> source/confidence warning
```

This is more realistic than trying to generate rich images.

## Phosphene glyphs

A phosphene glyph is a learned perceptual symbol produced by stimulation.

```text
Cognitive Packet: “low confidence”
Neural glyph: two dim pulses in upper-right field
Meaning learned by user: do not state this as fact
```

```text
Cognitive Packet: “ask a question”
Neural glyph: small curved trace
Meaning learned by user: turn the cue into a question
```

## Possible research prototype without implants

SYMBORG can simulate this today using AR or screen-based phosphene simulation.

```text
1. Convert packet into symbolic glyph.
2. Display glyph in AR/screen as phosphene-like dots.
3. Train user to recognize symbols.
4. Measure speed, accuracy, and cognitive load.
5. Later compare with BCI/visual prosthesis constraints.
```

This gives the project a credible bridge to visual neuroprosthetics without making surgical claims.

## Ethical boundary

Neural-write systems must be opt-in, clinically governed, and designed around autonomy.

Rules:

```text
No unconscious stimulation for persuasion.
No commercial advertising channel.
No unlogged stimulation.
No stimulation without explicit user intent.
No hidden modulation of emotion or belief.
```

## SYMBORG position

```text
Neuralink/visual prosthesis systems may build the neural hardware.
SYMBORG can design the cognitive output language, timing, and user agency layer.
```

This keeps SYMBORG complementary, not competitive.
