# SYMBORG Output Roadmap

This roadmap organizes feedback modes by feasibility and ambition.

## Version map

```text
v0  Screen cards
v1  Audio whisper + haptic control
v2  AR peripheral cards
v3  Physiology-aware adaptive delivery
v4  Assistive communication terminal
v5  BCI-compatible intent selection
v6  Neural-write research layer
```

## v0: Screen cards

Goal: prove cognitive packet usefulness.

Output:

```text
3 ranked packets
confidence score
expected use
source basis
```

## v1: Audio whisper + haptic control

Goal: make SYMBORG usable without looking at a screen.

Output:

```text
short voice cue
haptic status
ring control
```

## v2: AR peripheral cards

Goal: make contextual knowledge visible without stealing attention.

Output:

```text
one-line card
subtle icon
confidence/source marker
```

## v3: Adaptive delivery

Goal: stop overloading the user.

Output policy changes based on:

```text
pause length
user speaking
movement
stress/load estimate
environmental risk
```

## v4: Assistive communication terminal

Goal: support users with limited input channels.

Output:

```text
candidate sentence grid
confirm/cancel
speak aloud
save phrase
```

## v5: BCI-compatible selection

Goal: use decoded intent as an input signal.

Examples:

```text
select candidate A/B/C
confirm/cancel
open/close output
request depth
```

## v6: Neural-write research layer

Goal: explore direct sensory output as symbolic perceptual cues.

Do not attempt full internal screen reproduction.

Start with:

```text
phosphene-like AR simulation
symbol learning
low-bandwidth glyphs
closed-loop calibration concepts
```

## The key design shift

From:

```text
AI answer generator
```

To:

```text
human perception router
```
