# Product Roadmap

This roadmap is designed to keep the project credible. The path starts with non-invasive software and interaction design before moving toward wearables, AR, and future BCI-compatible assistive workflows.

## Phase 1: Concept and Research Foundation

Deliverables:

- One-page manifesto
- Whitepaper draft
- Architecture diagram
- Ethics principles
- Visual system
- Initial source list

Success criteria:

- The concept is clear without hype
- Claims are narrow and defensible
- The project reads as a serious product architecture

## Phase 2: Contextual Whisper Prototype

Deliverables:

- Transcript input
- Topic and entity detection
- Retrieval from a small source library
- Three ranked micro-cues
- Confidence score
- Simple web UI or CLI demo

Success criteria:

- Output arrives in under 2 seconds for small examples
- Cues are shorter than generated answers
- The user can ignore, deepen, or silence the assistant

## Phase 3: Cognitive Packet Engine

Deliverables:

- Packet type taxonomy
- Prompt and ranking rules
- Fact cue, question cue, counterpoint cue, summary cue
- Evaluation rubric for usefulness and interruption cost

Success criteria:

- The engine produces usable cues instead of paragraphs
- It can choose silence when confidence is low
- It can produce a source-aware explanation on request

## Phase 4: Local Memory Vault

Deliverables:

- Local note ingestion
- Personal reading list
- Meeting notes
- Simple source index
- Delete and export controls

Success criteria:

- User can see and manage stored context
- Retrieval is transparent
- Sensitive memory is not sent to unnecessary services

## Phase 5: Audio Whisper Mode

Deliverables:

- Short spoken cue output
- User-adjustable verbosity
- Silence command
- Meeting and conversation demo

Success criteria:

- The audio cue does not break conversational flow
- Users prefer short cue modes over full answers in live settings

## Phase 6: Haptic Control Prototype

Deliverables:

- Simulated ring controls
- Tap, double tap, long press, and swipe mappings
- Interaction latency testing

Success criteria:

- The user can control depth and silence without looking at a screen

## Phase 7: AR Mockup

Deliverables:

- Peripheral micro-cue UI
- Single-line cue design
- Field-of-view constraints
- Distraction testing plan

Success criteria:

- The UI supports attention instead of stealing it

## Phase 8: User Testing

Measures:

- Cue usefulness
- Interruption cost
- Conversation naturalness
- Factual accuracy
- Perceived agency
- Stress and overload

## Phase 9: Passive Load Sensing Exploration

Scope:

- Non-invasive and non-medical only
- Focus, overload, or interaction state estimation
- No thought-content claims

Success criteria:

- Load sensing improves silence and timing decisions
- The system avoids inferring sensitive mental content

## Phase 10: Assistive Communication Demo

Deliverables:

- Constrained-input sentence selection
- Three candidate utterances
- Context-aware ranking
- Switch, gaze, or keyboard simulation

Success criteria:

- Fewer input actions per useful sentence
- Clear improvement over generic phrase boards in selected scenarios

## Phase 11: Public Whitepaper and Demo Video

Deliverables:

- Polished whitepaper
- Two-minute demo video
- GitHub repository
- Professional landing page

## Phase 12: External Feedback

Targets:

- HCI researchers
- Assistive technology builders
- Neurotechnology ethicists
- Wearable AI founders
- BCI research communities
- Accessibility advocates
